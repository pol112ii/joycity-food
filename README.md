# 음식만들기 자동 온도조절

게임 "음식만들기" 미니게임의 온도계를 자동으로 적정 구간(눈금 두 선 사이)에
유지해주는 프로그램. 실제 플레이 영상을 프레임 단위로 분석해서 만들었음.

## 영상 분석으로 알아낸 것

| 항목 | 측정값 |
|---|---|
| 적정 구간 위치 | 관 위에서 42% ~ 54% 지점 (관 전체의 12%로 좁음) |
| 가열 속도 (+ 꾹) | 관 전체를 약 5초에 통과 → 적정 구간은 0.6초 만에 지나침 |
| 냉각 속도 (-) | 가열의 약 1/3 속도 |
| 관성 | 버튼을 떼도 관 높이의 6~7%만큼 더 움직임 |

그래서 이 봇은 **연타하지 않고**(연타 시 게임 튕김) 버튼을 **꾹 누른 채
온도를 실시간 감시하다가, 관성을 계산해서 목표보다 미리 떼는** 방식으로 동작함.

## 파일 구성

| 파일 | 용도 |
|---|---|
| `cook_bot.py` | 본체. F8 시작/정지, F9 종료 |
| `measure.py` | 좌표/색상 측정 도구 (마우스 올린 곳의 좌표와 RGB 출력) |
| `diagnose.py` | 진단 도구. 클릭 없이 봇의 온도계 인식 상태만 보여줌 |

## 설치 (윈도우)

```
python -m pip install -r requirements.txt
```

## 사용 순서

1. 게임 창을 항상 쓰는 자리에 고정 (옮기면 좌표 다시 측정해야 함)
2. `measure.py` 실행해서 아래 값 측정 후 `cook_bot.py` 상단 설정값 수정
   - 온도계 관 맨위/맨아래 좌표 → `GAUGE_LEFT/TOP/WIDTH/HEIGHT`
   - 빨간(+) 손잡이 중심 → `PLUS_BTN`
   - 파란(-) 손잡이 중심 → `MINUS_BTN`
   - 노란색/빨간색 수은 RGB → `YELLOW_RGB` / `RED_RGB`
3. `diagnose.py` 실행해서 게임 플레이하며 인식이 잘 되는지 확인
   (위치 숫자가 수은 움직임 따라 변하고, 상태 표시가 맞으면 OK)
4. `cook_bot.py` 실행 → 게임에서 요리 시작 → **F8** 눌러 봇 시작
5. 멈추려면 **F8**, 완전 종료는 **F9**, 비상시엔 마우스를 화면 왼쪽 위 구석으로

## 튜닝 가이드

`cook_bot.py` 상단 설정값:

- 온도가 구간을 자꾸 **위로 넘어감** → `MOMENTUM_UP`을 키우기 (0.06 → 0.09)
  = 더 일찍 떼게 됨
- 구간까지 **못 미치고 아래서 멈춤** → `MOMENTUM_UP`을 줄이기 (0.06 → 0.04)
- **아래로 너무 떨어짐** → `MOMENTUM_DOWN`을 키우기
- 수은을 **인식 못 함** (diagnose에서 노랑/빨강 0) → `COLOR_TOL` 키우기 (50 → 70),
  그래도 안 되면 `measure.py`로 색상 다시 측정
- 버튼 누르는 시간이 너무 길다/짧다 → `MAX_HOLD` 조절
- 버튼을 뗀 뒤 쉬는 시간 → `MIN_GAP` (너무 줄이면 연타처럼 되니 0.25 밑으로는 비추천)
